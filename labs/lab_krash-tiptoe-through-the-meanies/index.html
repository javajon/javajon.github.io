Tiptoe Through the Meanies

Challenge your Kubernetes skills with this puzzle.

This lab is part of a series that challenges your skills with running applications on Kubernetes. This particular lab focuses on mysterious application instability in a microservice architecture that seems to have no clear cause.

You are provided with a working Kubernetes cluster and a live satellite tracker application that monitors five popular satellites in real time. The application uses a microservice architecture with separate satellite tracking services and an aggregator frontend. Users report that individual satellites randomly disappear, data fails to load intermittently, and the system shows unpredictable failures. Your mission is to investigate these stability issues and restore reliable service.

In this lab, you will learn how to:

    ☐ Investigate unexplained pod failures in a StatefulSet
    ☐ Debug intermittent communication issues between microservices
    ☐ Analyze patterns in system-wide disruptions across services
    ☐ Identify and neutralize sources of instability in distributed systems
    ☐ Implement solutions to protect microservices from external interference

The symptoms are frustrating: satellite-info pods randomly die, HTTP requests between services mysteriously fail, and the application behaves erratically without any obvious configuration issues or resource constraints. Can you uncover what’s causing this chaos?

This lab features a Python Flask microservice application that tracks five satellites:

The live satellite tracker provides real-time position data for ISS, Hubble, NOAA-20, Landsat-9, and Starlink satellites. When working correctly, it displays all satellites on a live map with orbital paths. When disrupted, users see disappearing satellites, connection failures, and intermittent data loss.
