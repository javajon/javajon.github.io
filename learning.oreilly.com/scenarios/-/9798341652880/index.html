The Silent Stats

Challenge your Kubernetes debugging skills with this puzzle.

This lab is part of a series that challenges your skills with running applications on Kubernetes. Each lab presents you with a common problem often encountered when running applications on Kubernetes.

In this lab, you are provided with a working Kubernetes cluster. You will be challenged with an X-ray analysis application that appears to be running but isn’t functioning correctly. This lab will introduce you to more advanced debugging techniques that go beyond standard log analysis.

In this lab, you will learn how to:

    ☐ Debug applications that don’t provide useful logs in standard output
    ☐ Use the kubectl debug command to perform advanced troubleshooting
    ☐ Investigate system-level issues in a running container
    ☐ Understand how security contexts affect container behavior

Kubernetes applications can sometimes experience issues that aren’t immediately apparent through standard monitoring and logging. Good luck uncovering what’s happening behind the scenes!

This lab leverages a Python Flask application that simulates a medical X-ray analysis system:

The application processes radiological X-rays and sends them to an AI system for cancer detection analysis. It should generate metrics about the presence of cancer in the X-rays and display those metrics in a dashboard.
